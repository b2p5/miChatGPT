Descripción del proyecto AgramDAO.\
  AgramDAOs se define como una incubadora de DAOs en Bitcoin. \
  Una DAO es una Organización Autónoma Descentralizada.\
  Utiliza la tecnología Agram Project para su desarrollo y proporciona \
  un modelo de plantillas para posteriores desarrollos de proyectos \
  informáticos basados en la soberanía, la inmutabilidad y la trazabilidad. \
  \
  \
Partes para el desarrollo de un proyecto en AgramDAO:\
  1. -Descripción del proyecto.\
  2. -Captación de colaboradores.\
  3. -Estudio previo - Contrato inteligente.\
  4. -Desarrollo del software con Agram Project.\
  5. -Pruebas de funcionamiento e implantación.\
  6. -Marketing en plataformas.\
  7. -Gestión de Ventas y Beneficios.\

Explicación de las partes de un proyecto en AgramDAO:\
1. -Descripción del proyecto:\
  El creador del proyecto proporciona una breve descripción del mismo\
  y asigna una fecha de finalización y/o objetivos a conseguir y un \
  porcentaje sobre los futuros beneficios obtenidos para cada una de \
  las partes del desarrollo.\
 \
2. -Captación de colaboradores.\
  A continuación se publica dicho proyecto en una serie de canales para\
  llevar a cabo la captación de colaboradores\
  Las personas interesadas en colaborar en el proyecto se auto asignan \
  la tarea/s en la que quieren participar. \    
  Se establecerán varios canales de comunicación, mensajes, \
  reuniones, llamadas, etc para resolver dudas, perfilar detalles y\
  determinar los deberes y derechos por parte de todos los componentes \
  del equipo. \
  \
3. -Estudio previo - Contrato inteligente.\
  En este punto se establece, mediante contrato inteligente,\
  todos los puntos a cumplir por cada uno de los componentes del \
  equipo así como los repartos automáticos de los beneficios obtenidos\
  por la aplicación.\
  \
4. -Desarrollo del software con Agram Project.\
  Cerrado el Estudio previo, se comienza con el desarrollo del software.\
  Este se hará con Agram Project. \
  \
5. -Pruebas de funcionamiento e implantación.\
  A medida que se van concluyendo el desarrollo de los distintos módulos\
  del proyecto el 'Equipo de pruebas de funcionamiento' realizará aquellas \
  pruebas que se le hayan asignado mediante contrato. Finalizado el \
  desarrollo de pasará a los servidores de explotación.\
  \
6. -Marketing en plataformas.\  
  El Equipo de Marketing promocionará la aplicación en aquellos canales \
  establecidos previamente en el contrato inteligente.\
  \
7. -Gestión de Ventas y Beneficios.\
  Finalmente, a partir de lo establecido en el contrato inteligente, \
  la aplicación proporcionará informes de Ingresos - Gastos, Beneficios\
  consolidados y Reparto (según determine el contrato inteligente) de dichos beneficios.\
\
\
Funcionamiento interno de la aplicación:\
  Una vez que el proyecto esté en explotación, de cada transacción \
  económica realizada dentro del mismo se detraerá un porcentaje \
  como pago por el uso de la aplicación. Este porcentaje se acuerda \
  entre el creador de la aplicación y los colaboradores y formará parte \
  del contrato inteligente. Dicho monto pasará a formar parte de las ganancias
  de la aplicación que servirán tanto para el mantenimiento de esta 
  como para hacer los pagos a los miembros del equipo (creador del proyecto y colaboradores).\
  \
  El nombre del proyecto contiene la palabra DAO porque su objetivo \
  es descentralizar las siguientes partes: \
   -Las tareas que forman parte del desarrollo, \
   -Los acuerdos a los que se hayan llegado entre los participantes y \
    que se encuentren registrados en el contrato inteligente firmado al \
    concluir el Estudio Previo  
   -Las posteriores tareas  gestión de Ingresos - Gastos - Reparto de Beneficios.\
    \
  Todas las operaciones que se realizan internamente quedan registradas \
  mediante NFTs en varias cadenas de bloques,dos de ellas se encuentran dentro del protocolo \
  de Agram Project y de la consolidación de todos los NFTs diarios se genera un Hash el cual \
  se grabará en la cadena de bloques de Bitcoin, lo que le confier al todo el proceso \
  inmutabilidad y trazabilidad.\
